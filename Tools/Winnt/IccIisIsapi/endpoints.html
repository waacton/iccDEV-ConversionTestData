  • The upload workflow persists uploads, generated files, and logs into /_tool-work/.
  • The workspace root now has its own landing page, so it avoids IIS directory-level 403.14.

Run The Top 4 Tools In The Browser

Upload an ICC profile or ICC XML file and the IIS endpoint will run the tool suite on the server: iccToXml, iccFromXml, iccDumpProfile, and iccRoundTrip.

ICC upload flow: iccToXml -> iccFromXml -> iccDumpProfile -> iccRoundTrip. XML upload flow starts with iccFromXml and then runs the remaining tools on the regenerated ICC profile.

Each request also gets a persisted workspace page under ./_tool-work/<job>/, with direct links to the uploaded input, logs, and generated artifacts.

Response Expectations

  • health should return ok with text/plain; charset=utf-8.
  • summary should include both IccProfLib and IccLibXML version strings.
  • xml should return application/xml; charset=utf-8 and begin with an XML declaration.
  • tools should return application/json; charset=utf-8 and one result card per wrapped tool.

Next Step

If the responses are not what you expect, switch to Integration Guide and re-check the site root, handler verbs, runtime DLL set, and persisted workspace behavior.
